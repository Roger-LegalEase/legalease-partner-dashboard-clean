## Method

The audit mirrors the current consumer screen derivation: sort profile questions by `flowStages` stage order and original profile order, and exclude only `source_question_*` rows. It reads `src/lib/rcap-engine/compiled/all51.json` as the current designer public projection source and cross-checks `src/lib/rcap-engine/compiled/profiles/*.json` to ensure all 51 engine profiles are present.

Classification is conservative. A question is only proposed for later post-payment movement when its id/prompt looks like court, county, case number, charge, document, or filing-logistics collection. Review/caution candidates are not recommendations to allow payment; they are candidates for later source-reviewed caution handling only when all hard stops, timing blocks, exclusions, automatic/no-filing routes, and true pathway facts remain satisfied.

## Guardrails

Payment must remain blocked for `hard_stop`, `guidance_only`, `not_yet`, `likely_not_eligible`, `needs_review`, and missing eligibility facts.

- `hard_stop`: the matter is outside self-help state screening scope, such as not the user's own record or a federal/out-of-scope matter.
- `guidance_only`: the selected route is automatic, no-filing, or guidance-only, so charging for a filing packet would be unsafe and misleading.
- `not_yet`: a source-defined timing, waiting-period, completion, supervision, or pending-case condition is not satisfied.
- `likely_not_eligible`: the user selected a source-defined exclusion or disqualifying/review-required category.
- `needs_review`: the supplied facts are ambiguous, unknown, or do not match an executable source-defined route.
- Missing true eligibility or pathway facts: the evaluator currently returns `needs_more_info` for required public facts. Any future relaxation must distinguish packet-preparation details from facts that drive scope, pathway, exclusions, timing, automatic/no-filing routes, or legal result code.

Payment should remain allowed only when the evaluator reaches `packet_ready` or `packet_ready_with_caution` and the packet plan remains fulfillment-ready. This audit does not recommend payment for any other result code.
